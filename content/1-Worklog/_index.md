**On this page**, you'll find the complete worklog from my internship at FCAJ. The journey started from the most basic cloud concepts and ended with a fully deployed **Flash Sale Ticket Booking System** on AWS.

12 weeks total, split into 3 clear phases: learning fundamentals → designing architecture → building a real product.

---

**Week 1:** [Created AWS account, explored the Console, learned about EC2, and ran a backend with PM2.](1.1-week1/)

**Week 2:** [Studied compute services, networking, VPC, Security Groups, and created a first Lambda function.](1.2-week2/)

**Week 3:** [Learned Storage (S3, EBS, EFS) and Databases (RDS, DynamoDB), hands-on with S3.](1.3-week3/)

**Week 4:** [Explored AI/ML on AWS, IAM, and security — practiced access control on the Console.](1.4-week4/)

**Week 5:** [Researched High Availability, Microservices, Event-Driven Architecture, and attended AWS Community Day.](1.5-week5/)

**Week 6:** [Studied Migration, the Well-Architected Framework (6 pillars), and practiced with AWS Pricing Calculator.](1.6-week6/)

**Week 7:** [Deployed a Node.js app to Elastic Beanstalk and explored CI/CD with CodePipeline.](1.7-week7/)

**Week 8:** [Designed an async processing flow with SQS + SNS, configured DLQ, and ran real tests.](1.8-week8/)

**Week 9:** [Studied Amazon RDS, connected applications to the database, practiced backup and database security.](1.9-week9/)

**Week 10:** [Built an online football field booking web app on AWS.](1.10-week10/)

**Week 11:** [Continued developing and completing the remaining project features.](1.11-week11/)

**Week 12:** [Finalized the product, wrote the internship report, and submitted the project.](1.12-week12/)
