**On this page**, you will find the complete worklog of our internship project. We spent the internship learning AWS concepts and eventually implementing the Flash Sale Ticket Booking System.

The worklog spans 12 weeks, covering everything from AWS fundamental training to the final CI/CD and production deployment:

**Week 1:** [Learned the basics of AWS, AWS Console, and AWS CLI.](1.1-week1/)

**Week 2:** [Explored the integration between AWS Console and AWS CLI for resource management.](1.2-week2/)

**Week 3:** [Studied AWS Compute, Storage, Networking, and Database services; created an AWS Free Tier account and practiced AWS CLI commands.](1.3-week3/)

**Week 4:** [Practiced launching EC2 instances, connecting via SSH, attaching EBS volumes, and managing AWS resources using the Console and CLI.](1.4-week4/)

**Week 5:** [Worked with AWS Lambda, Amazon CloudWatch, Grafana, Resource Groups, Tags, and IAM.](1.5-week5/)

**Week 6:** [Studied AWS CAF, Migration strategies, AWS Well-Architected Framework, and AWS Pricing Calculator; reviewed Cloud Practitioner knowledge.](1.6-week6/)

**Week 7:** [Learned AWS Elastic Beanstalk, deployed a Node.js application, and explored AWS CI/CD services (CodeCommit, CodeBuild, and CodePipeline).](1.7-week7/)

**Week 8:** [Studied Event-Driven Architecture and implemented Amazon SQS and Amazon SNS for asynchronous processing.](1.8-week8/)

**Week 9:** [Learned Amazon RDS, connected applications to the database, and practiced backup, monitoring, and database security.](1.9-week9/)

**Week 10:** [Developed an online football booking web application using AWS services.](1.10-week10/)

**Week 11:** [Continued developing and completing the remaining features of the project.](1.11-week11/)

**Week 12:** [Finalized the project, prepared the internship report, and submitted the completed product.](1.12-week12/)
